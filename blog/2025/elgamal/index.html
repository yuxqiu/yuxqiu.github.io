ElGamal Encryption

tl;dr: I’ve got nothing1, except Alin Tomescu’s write-up2.

I will elaborate a bit on why ElGamal public key encryption can be thought of as approximately equivalent to using an ephemeral Diffie-Hellman (DH) exchanged key as a one-time pad.

Prerequisite

Diffie-Hellman (DH) Key Exchange

  1. Let GGG be a cyclic group of prime order qqq with generator ggg.
  2. Party A:
    • Picks a secret key a∈Zqa \in \mathbb{Z}_qa∈Zq​.
    • Computes the public key: pkA=ga\textcolor{teal}{pk_A} = g^apkA​=ga.
  3. Party B:
    • Picks a secret key b∈Zqb \in \mathbb{Z}_qb∈Zq​.
    • Computes the public key: pkB=gb\textcolor{orange}{pk_B} = g^bpkB​=gb.
  4. Both parties compute the shared key:
    • A computes: (pkB)a=gba(pk_B)^a = g^{ba}(pkB​)a=gba.
    • B computes: (pkA)b=gab(pk_A)^b = g^{ab}(pkA​)b=gab.

Both parties arrive at the same shared secret gab\textcolor{#8B4513}{g^{ab}}gab, which can be used for symmetric encryption.

See also: Ephemeral and/or static keys.

Original ElGamal Encryption

Let’s first recall the original ElGamal Encryption. Let GGG be a cyclic group of prime order qqq with generator ggg. An ElGamal Encryption is a tuple of 3 PPT algorithms:

  1. KeyGen:
    • Choose a random secret key sk∈Zqsk \in \mathbb{Z}_qsk∈Zq​.
    • Compute the public key pk=gskpk = g^{sk}pk=gsk.
    • Output: (pk,sk)(pk, sk)(pk,sk).
  2. Encrypt(pk, m):
    • Choose a random r∈Zqr \in \mathbb{Z}_qr∈Zq​.
    • Compute c1=grc_1 = g^rc1​=gr.
    • Compute c2=m⋅pkr=m⋅gr⋅skc_2 = m \cdot pk^r = m \cdot g^{r \cdot sk}c2​=m⋅pkr=m⋅gr⋅sk.
    • Output: ciphertext c=(c1,c2)c = (c_1, c_2)c=(c1​,c2​).
  3. Decrypt(sk, c):
    • Parse ciphertext c=(c1,c2)c = (c_1, c_2)c=(c1​,c2​).
    • Compute shared secret s=c1sk=gr⋅sks = c_1^{sk} = g^{r \cdot sk}s=c1sk​=gr⋅sk.
    • Recover message: m=c2⋅s−1m = c_2 \cdot s^{-1}m=c2​⋅s−1.

So, why does ElGamal Encryption resemble an Ephemeral DH Key Exchange?

Suppose A wants to send a message to B and knows B’s ElGamal public key. In this scenario:

  • B’s public key in ElGamal encryption is essentially a static DH key, written as gskB\textcolor{orange}{g^{sk_B}}gskB​.
  • When A encrypts a message using B’s public key, it effectively performs a DH key exchange:
    • A picks a random rrr and computes gr\textcolor{teal}{g^r}gr, an ephemeral DH key.
    • A then computes a shared key gr⋅skB\textcolor{#8B4513}{g^{r \cdot sk_B}}gr⋅skB​ and uses it to blind the message mmm.

What are the differences?

  • It’s not a pure One-Time Pad (OTP) because the encryption uses multiplication rather than XOR3.
