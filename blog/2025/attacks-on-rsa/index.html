Attacks on RSA

tl;dr: I discuss the issues related to small exponents, how to improve the success rate of an attack on RSA, how the square root relates to the hardness of the RSA problem, and its connection with Shor’s algorithm. Many of the materials discussed below are thanks to the lecture notes and coursework that I have taken in Introduction to Cryptography.

Prerequisites

  • RSA
  • Chinese Remainder Theorem (CRT)
  • Shor’s Algorithm (for the last section only)

Small Exponent Attack

Alice wants to send a (k − 1)-bit message, m m m, to Bob, Cathy, and Dave. Bob, Cathy, and Dave have distinct k-bit RSA keys, NB N_B NB​, NC N_C NC​, and ND N_D ND​, and they all use the same exponent, e=3 e = 3 e=3. Alice uses plain textbook RSA to encrypt m m m under each key, resulting in three ciphertexts cB c_B cB​, cC c_C cC​, and cD c_D cD​, which she sends to them. Eve intercepts all three ciphertexts. Show that Eve can recover the plaintext m m m.

There are two cases to consider:

  1. NA N_A NA​, NB N_B NB​, and NC N_C NC​ are not pairwise coprime.

    Without loss of generality, suppose NA N_A NA​ and NB N_B NB​ are not coprime. Then one can use gcd⁡(NA,NB) \gcd(N_A, N_B) gcd(NA​,NB​) to factor both NA N_A NA​ and NB N_B NB​. Once factored, computing ϕ(NA) \phi(N_A) ϕ(NA​) is straightforward, which then enables decryption of the ciphertext me mod NA m^e \bmod N_A memodNA​.

  2. NA N_A NA​, NB N_B NB​, and NC N_C NC​ are pairwise coprime.

    In this case, we have:

    cA=m3 mod NAcB=m3 mod NBcC=m3 mod NC\begin{align*} c_A &= m^3 \bmod N_A \\ c_B &= m^3 \bmod N_B \\ c_C &= m^3 \bmod N_C \\ \end{align*}cA​cB​cC​​=m3modNA​=m3modNB​=m3modNC​​

    Using the Chinese Remainder Theorem (CRT), we can compute a value c′ c' c′ such that

    c′=m3 mod (NANBNC)c' = m^3 \bmod (N_A N_B N_C)c′=m3mod(NA​NB​NC​)

    Since m m m is smaller than any of the moduli, m3<NANBNC m^3 < N_A N_B N_C m3<NA​NB​NC​. Therefore, the cube root of c′ c' c′ can be taken directly to recover m m m.

Improve RSA Attack Success Rate

Let (N,e) (N, e) (N,e) be an RSA key, where 2<e<N 2 < e < N 2<e<N. Suppose A \mathcal{A} A is a probabilistic algorithm that runs in time t t t with

Pr⁡r←ZN[A(N,e,re mod N)=r]=1%.\Pr_{r \leftarrow \mathbb{Z}_N} \left[ \mathcal{A}(N, e, r^e \bmod N) = r \right] = 1\%.r←ZN​Pr​[A(N,e,remodN)=r]=1%.

Show that A \mathcal{A} A can be used as a subroutine to construct an algorithm B \mathcal{B} B such that

Pr⁡r←ZN[B(N,e,re mod N)=r]=99%.\Pr_{r \leftarrow \mathbb{Z}_N} \left[ \mathcal{B}(N, e, r^e \bmod N) = r \right] = 99\%.r←ZN​Pr​[B(N,e,remodN)=r]=99%.

To build such an algorithm B \mathcal{B} B, we rely on the fact that RSA is multiplicatively homomorphic. With this in mind, one can devise an algorithm B(N,e,y) \mathcal{B}(N, e, y) B(N,e,y) as follows:

  • Repeat:
    • Sample s←$ZN s \xleftarrow{\$} \mathbb{Z}_N s$​ZN​.
    • Compute x←A(N,e,yse mod N) x \leftarrow \mathcal{A}(N, e, ys^e \bmod N) x←A(N,e,ysemodN).
  • Until xe=yse x^e = ys^e xe=yse.

Then, by using the homomorphic property, recover r r r as:

r←x⋅s−1 mod N.r \leftarrow x \cdot s^{-1} \bmod N.r←x⋅s−1modN.

By repeating the algorithm A \mathcal{A} A a bounded number of times, we can design B \mathcal{B} B with a success probability of 99%.

Square Root and RSA

Let’s begin by examining an interesting attack.

RSA Key Leakage Vulnerability

Suppose Bob uses the RSA cryptosystem and has a key pair with public key e e e and private key d d d. If Bob accidentally leaks his private key and decides to generate a new public/private key pair without generating a new modulus, show that this approach is insecure.

To prove that this approach is insecure, we will show that, given d d d, one can factor N N N (assuming N N N is the product of two odd primes).

  1. Given d d d, compute k=ed−1 k = ed - 1 k=ed−1 using the public key e e e.
  2. By definition, k k k is divisible by ϕ(N) \phi(N) ϕ(N). Since Euler’s totient function ϕ(N) \phi(N) ϕ(N) is even, k k k is even, and hence k2 \frac{k}{2} 2k​ is an integer.
  3. By Euler’s theorem, for every g∈ZN∗ g \in \mathbb{Z}_N^* g∈ZN∗​, we have gk=1 g^k = 1 gk=1. Therefore, gk2 g^{\frac{k}{2}} g2k​ is a square root of 1 mod N 1 \bmod N 1modN.

Shor’s Algorithm

The idea behind Shor’s algorithm (period finding) is also connected to using square roots to factor N N N.

Suppose we use Shor’s algorithm to find an r r r such that

f(x+r)=f(x),f(x + r) = f(x),f(x+r)=f(x),

that is,

(gx)r=gx⇒gr−1≡1 mod N.(g^x)^r = g^x \quad \Rightarrow \quad g^{r-1} \equiv 1 \bmod N.(gx)r=gx⇒gr−1≡1modN.

Then, by the previous arguments, with probability at least 12 \frac{1}{2} 21​, we can use gr−12, …, gr−12l g^{\frac{r-1}{2}},\ \dots,\ g^{\frac{r-1}{2^l}} g2r−1​, …, g2lr−1​ to factor N N N. Consequently, one can repeatedly sample elements and apply Shor’s algorithm until success is achieved.

Another perspective on how a nontrivial square root aids in factoring N N N is to note that for an element with gr−1≡1 mod N g^{r-1} \equiv 1 \bmod N gr−1≡1modN, we have:

(gr−12−1)(gr−12+1)≡0 mod N.\left(g^{\frac{r-1}{2}} - 1\right) \left(g^{\frac{r-1}{2}} + 1\right) \equiv 0 \bmod N.(g2r−1​−1)(g2r−1​+1)≡0modN.

Thus, one of the factors in the parenthesis will be a prime factor of N N N (provided gr−12≠±1 g^{\frac{r-1}{2}} \neq \pm 1 g2r−1​=±1).
